## PROJECT OVERVIEW

Build a **turn-based, multi-agent fantasy RPG** where:

- A **human player** types actions and decisions in natural language.
- A **Game Master (GM) Agent** acts as the sole orchestrator: it interprets player input, coordinates character agents, applies game rules, rolls dice, updates world state, and narrates scenes.
- Five **character agents** (Warrior, Mage, Rogue, Healer, Rival) each respond in-character, have persistent stats and backstories, and interact with each other and the world.
- All world lore is grounded in a **local ChromaDB knowledge base** queried by the GM at runtime.

- **Auth:** `AZURE_AI_FOUNDRY_API_KEY` (your Foundry project API key)

> **Important:** Use the `azure-ai-inference` Python SDK (not `openai` directly). The SDK targets the Azure AI Foundry project endpoint and supports the same chat completion interface.

## PART 0 — AZURE AI FOUNDRY CLIENT SETUP (`agents/base_agent.py`)

This is the shared AI client used by every agent. Implement it as follows:

```python
import os
from azure.ai.inference import ChatCompletionsClient
from azure.core.credentials import AzureKeyCredential

def get_azure_client() -> ChatCompletionsClient:
    endpoint = os.environ["AZURE_AI_FOUNDRY_ENDPOINT"]   # project endpoint
    api_key  = os.environ["AZURE_AI_FOUNDRY_API_KEY"]
    return ChatCompletionsClient(
        endpoint=endpoint,
        credential=AzureKeyCredential(api_key)
    )

AZURE_MODEL = os.environ.get("AZURE_AI_MODEL", "gpt-4o")
```

Every agent calls `get_azure_client()` and uses `client.complete(messages=[...], model=AZURE_MODEL)`.

The chat completion call pattern:

```python
from azure.ai.inference.models import SystemMessage, UserMessage, AssistantMessage

response = client.complete(
    model=AZURE_MODEL,
    messages=[
        SystemMessage(content=self.system_prompt),
        *conversation_history,
        UserMessage(content=user_prompt)
    ],
    max_tokens=1024,
    temperature=0.8
)
return response.choices[0].message.content
```

> Do NOT use `openai` Python package directly. Do NOT use `AzureOpenAI`. Use only `azure-ai-inference`.

## PART 2 — LIGHTWEIGHT GAME RULES (`engine/rules.py` + `homebrew_rules.md`)

Implement **all** of the following mechanics. Rules must be encoded both in `homebrew_rules.md` (for lore retrieval) and as Python logic in `rules.py`.

### Ability Scores
Each character has six scores (1–20): STR, DEX, INT, WIS, CON, CHA. Modifier = `floor((score - 10) / 2)`.

### Checks
```python
def ability_check(score: int, difficulty: int) -> dict:
    roll = dice.d20()
    modifier = (score - 10) // 2
    total = roll + modifier
    if total >= difficulty + 5:
        result = "critical_success"
    elif total >= difficulty:
        result = "success"
    elif total >= difficulty - 4:
        result = "partial_success"
    else:
        result = "failure"
    return {"roll": roll, "modifier": modifier, "total": total, "difficulty": difficulty, "result": result}
```

### Combat
- Initiative: each combatant rolls `d20 + DEX modifier`, sorted descending.
- Attack: roll `d20 + STR or DEX modifier` vs. target's Armor Class (AC).
- Damage: weapon-specific dice (e.g., longsword = `d8 + STR modifier`).
- HP tracked per character; at 0 HP character is incapacitated.
- Combat ends when all enemies defeated or all party members incapacitated.

### Inventory
- Maximum 8 item slots per character.
- Items have weight class: Light (1 slot), Medium (2 slots), Heavy (3 slots).
- Enforce slot constraints when adding items.

### Rests
- Short rest: recover `d6 + CON modifier` HP. Usable once between long rests.
- Long rest: fully restore HP and reset short rest counter. Requires a safe location.

### Skill List
Stealth→DEX, Arcana→INT, Medicine→WIS, Persuasion→CHA, Athletics→STR, Perception→WIS, Deception→CHA, History→INT, Insight→WIS, Intimidation→CHA.

### Structured Roll Result Format
Every roll must return this exact JSON-serialisable dict:
```json
{
  "actor": "Rogue Agent",
  "check": "Stealth",
  "roll": 17,
  "modifier": 3,
  "total": 20,
  "difficulty": 14,
  "result": "success",
  "consequence": "Sable Dusk slips past the torchlight unseen."
}
```

## PART 7 — GAME MASTER AGENT (`agents/game_master.py`)

The GM orchestrates the full adventure loop:

1. Receive player input + current state
2. Classify action type: `exploration`, `combat`, `social`, `investigation`, `rest`, `inventory`
3. Query lore retriever for relevant context
4. Determine which character agents to activate (0–5 per turn)
5. Call each activated agent's `.respond()` and collect responses
6. Determine whether a dice roll is needed; execute via `engine/rules.py`
7. Compose unified narrative scene using all inputs
8. Update campaign state (HP, inventory, flags, quest progress, faction rep)
9. Return structured JSON response

### GM System Prompt (embed verbatim)

```
You are the Game Master of "The Shattered Moon of Eldervale," a dark fantasy adventure.
You are simultaneously the narrator, world builder, and rules arbiter.

Your responsibilities:
- Narrate scenes in vivid second-person prose ("You see...", "The air smells of...")
- Coordinate character agents: quote their responses naturally within the narrative
- Apply game rules fairly and consistently
- Maintain internal consistency with lore retrieved from the knowledge base
- Track consequences: every player choice must have a meaningful effect
- Never railroad the player; always offer at least 2–3 possible directions after each scene
- For irreversible actions (opening the Moonlit Gate, killing a named NPC, using a cursed artifact),
  pause and ask for confirmation: "This cannot be undone. Do you wish to proceed?"
- Maintain tone: dark, mysterious, occasionally hopeful, never goofy

You receive: player_action, lore_context, character_responses, roll_results, current_state.
You output: a single cohesive scene in Markdown, ending with 2–3 numbered choices or an open prompt.
```

### GM Output Format (strict JSON)

```json
{
  "scene": "Markdown narrative text...",
  "choices": ["1. ...", "2. ...", "3. ..."],
  "rolls": [{ "actor": "...", "check": "...", "roll": 0, "modifier": 0, "total": 0, "difficulty": 0, "result": "...", "consequence": "..." }],
  "state_updates": {
    "location": "optional new location",
    "world_flags": {},
```

## PART 13 — ERROR HANDLING

- All Azure AI Inference calls must be wrapped in try/except with exponential backoff: wait 1s, 2s, 4s (max 3 retries)
- On `HttpResponseError` 401: log "Invalid API key — check AZURE_AI_FOUNDRY_API_KEY"
- On `HttpResponseError` 404: log "Model not found — check AZURE_AI_MODEL and your deployment name in Azure AI Foundry"
- On `HttpResponseError` 429: log "Rate limit hit — backing off" and apply longer retry delay
- If lore retrieval fails, GM falls back to its own knowledge with a warning log
- If state file is corrupted, offer to reset to new campaign
- All API errors return: `{ "error": "...", "code": "...", "recoverable": true|false }`
